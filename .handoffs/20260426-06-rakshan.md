## What Was Built This Session

### M5 — Sarvam auto-translation ✓
- `backend/services/translate.py` — `async translate(text, src, tgt)` using provider from config.yaml
  - Supports: `sarvam` (Sarvam AI), `google` (Cloud Translation v2), `openrouter` (any chat model)
  - Reads API key from environment (loaded via python-dotenv from `backend/.env`)
- `backend/routers/batch.py` — `POST /api/v1/batch/translate`
  - Body: `{ "youtube_id": "...", "force": false, "concurrency": 5 }`
  - Translates all segments where `en_auto` is null (or all if force=true)
  - Runs up to N concurrent API calls (semaphore-bounded asyncio.gather)
  - Sets `en_final = en_auto` where `en_human` is not set
  - Returns `{ translated, skipped, errors, message }`
- `backend/.env.example` — copy to `backend/.env` and fill `SARVAM_API_KEY`
- Frontend: **"✦ Auto-translate"** button (purple) in editor header
  - Shows success banner with count, or error with hint to check .env
  - Refreshes the page after completion so new translations appear

**To activate Sarvam:**
1. Sign up at https://dashboard.sarvam.ai (free tier available)
2. Copy your API key
3. Create `backend/.env` with: `SARVAM_API_KEY=your_key_here`
4. Restart uvicorn
5. Open editor → click "✦ Auto-translate"

### M7 — Fine-tuning export formats ✓
- `backend/routers/export.py` — added `?format=raw|alpaca|openai` to JSONL export
  - `raw` (default): `{"te": "...", "en": "...", "source": "youtube_id", "t": 12.34}`
  - `alpaca`: `{"instruction": "Translate...", "input": "te text", "output": "en text"}`
