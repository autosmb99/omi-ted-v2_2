## Remaining Modules

### M5 — Sarvam auto-translation (build this first)

**Why**: YouTube's `en_auto` is rough. Sarvam AI is purpose-built for Indian languages. Better auto-translation = less editing work per segment.

**What to build:**
- `backend/services/translate.py` — `async def translate(text: str, src: str, tgt: str) -> str` using provider from `config.yaml` (`sarvam` | `google` | `openrouter`)
- `backend/routers/batch.py` — `POST /api/v1/batch/translate` — fills `en_auto` for all null segments in a video, updates `en_final` where `en_human` is not set
- Frontend: "Auto-translate" button on editor page header

**Sarvam API**: https://www.sarvam.ai — free tier available. Key goes in `backend/.env` as `SARVAM_API_KEY`.

**Config routing** (already in `backend/config.yaml`):
```yaml
llm:
  provider: sarvam   # sarvam | google | openrouter | local
```

### M6 — Glossary enforcement

**Why**: Theological terms (దేవుడు=God, పరిశుద్ధాత్మ=Holy Spirit, etc.) must be consistent across all segments for fine-tuning quality.

**What to build:**
- `frontend/app/glossary/page.tsx` — manage term pairs (te_term, en_term, category)
- Backend: `GET/POST/DELETE /api/v1/glossary` endpoints in `videos.py`
- Editor: highlight Telugu text where a glossary term appears; warn if `en_human` uses a non-canonical translation
- Seed data: ~30 common Telugu Christian theology terms

### M7 — Fine-tuning export formats

**Why**: Current JSONL is raw parallel pairs. Modern fine-tuning needs instruction format.

**What to build** — add `?format=` param to `/api/v1/export/jsonl`:
- `format=raw` (default): `{"te": "...", "en": "..."}`
- `format=alpaca`: `{"instruction": "Translate this Telugu Christian sermon text to English.", "input": "...", "output": "..."}`
- `format=openai`: `{"messages": [{"role": "system", "content": "You are a Telugu to English translator specializing in Christian theology."}, {"role": "user", "content": "..."}, {"role": "assistant", "content": "..."}]}`
- Frontend: format selector dropdown on export buttons

### M8 — Dataset statistics page

**Why**: Know when you have enough data to fine-tune.

**What to build:**
- `frontend/app/stats/page.tsx` — summary cards: total segments, reviewed %, estimated token count, per-video breakdown table
- `GET /api/v1/stats` backend endpoint
